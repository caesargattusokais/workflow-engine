# v2.0.0+ 扩展路线图

## 高价值

- [ ] **数据库持久化** — 当前全内存，重启即丢。给 SPI 加 JDBC/MySQL 实现，草稿、定义、实例落库
- [x] **流程版本管理** — 定义支持多版本，实例绑定版本号。修改流程后新实例跑新版，老实例继续跑老版
- [ ] **定时任务/定时器** — Timer 节点：超时自动驳回、延迟触发、定时提醒
- [ ] **消息/信号事件** — 外部事件驱动流程：收到付款回调→自动通过、Kafka 消息→启动流程
- [ ] **表单绑定** — UserTask 绑定表单 schema，审批人看到带字段的表单而非光秃秃的按钮
- [ ] **通知集成** — 任务创建/完成/超时发送通知：邮件、钉钉、企微、站内信

## 中等价值

- [ ] **子流程 (CallActivity)** — 一个流程调用另一个流程，复用时不重复画
- [ ] **Saga 补偿回滚** — 服务任务失败，自动逆序调用补偿逻辑回滚已执行节点
- [ ] **流程导入导出** — Designer 导入/导出 YAML/JSON 文件，跨环境迁移
- [ ] **组织架构集成** — assignee/candidateGroups 对接 LDAP/企业组织树，动态解析审批人
- [ ] **SLA 监控** — 每个节点设超时时间，超时自动告警或升级
- [ ] **流程模板市场** — 预置常用模板（请假、报销、采购），一键导入

## 低价值

- [ ] **BPMN 2.0 支持** — 导入/导出标准 BPMN XML，兼容 Camunda Modeler
- [ ] **多语言 i18n** — 界面国际化
- [ ] **暗色/亮色主题切换**
- [ ] **移动端适配**
- [ ] **协作编辑** — 多人同时编辑同一草稿（OT/CRDT）
